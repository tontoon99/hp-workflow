# 運用フロー（名刺撮影から課金まで）

## 0. 現場（交流会）
1. 担当者がWantedly Peopleアプリを開く
2. 名刺を撮影（表裏）
3. OCR結果をその場で最低限修正
4. 連絡許諾メモ（任意）をタグで付与

利用ツール:
- Wantedly People（名刺撮影/OCR）

## 1. 取り込み（当日 or 翌日）
1. People PremiumからCSVをエクスポート
2. n8n WebhookへCSVアップロード
3. `company-record` を作成し重複排除

利用ツール:
- n8n（Webhook, Code, HTTP Request）
- 自作API（company-record保存）

## 2. 無料の比較ページ作成（支払い前）
1. 会社HPを特定（確信度付き）
2. 既存情報だけで3パターンのラフ案を作る
3. 比較ページURLを相手に送る
4. 「気に入れば本分析へ進む」案内をつける

利用ツール/API:
- Website解決API（自作）
- LLM API（ラフ案文面生成）
- テンプレート生成サービス（簡易版）

## 3. 支払い
1. 相手が比較ページ内の「本分析を依頼」ボタンを押す
2. Stripe Payment Linkへ遷移して決済
3. Stripe Webhookで `checkout.session.completed` を受信
4. 決済完了後に詳細分析を開始

利用ツール/API:
- Stripe Checkout
- Stripe Webhook API
- n8n Webhook Trigger

## 4. 有料詳細分析（決済後）
1. Playwrightで主要ページを取得
2. PSI API（mobile/desktop）実行
3. Lighthouse runnerで監査
4. 固定コンテンツJSONを生成

利用ツール/API:
- Playwright
- Google PageSpeed Insights API
- Lighthouse（外部worker）

## 5. 3案生成
1. 信頼/先進/親しみのbrief生成
2. デザイントークンを生成
3. テンプレA/B/Cへ流し込み（最終版）
4. 比較ページを本番品質で更新

利用ツール/API:
- LLM API（brief/token生成）
- AstroまたはNext.jsビルド環境

## 6. 品質レビュー
1. 自動品質ゲート（Lighthouse、リンク、見出し、content lock）
2. 人レビュー（R3）で文脈チェック
3. OKなら比較ページ作成

利用ツール:
- 自作QAサービス
- n8n Manual Approval

## 7. 納品・送信
1. 比較ページURLを1本発行
2. 3案URLと差分スコアを表示（決済済み相手のみ）
3. メール送信（停止導線を明示）

利用ツール:
- 自作proposalサービス
- メール配信サービス（SendGrid等）
